#### 3.1 Requisitos de Interfaces Externas

| ID | Requisito | Descripción | Prioridad |

| **MT5-02** | Datos de Mercado | El EA debe obtener precios en tiempo real (tick, OHLCV) y datos históricos a través de las funciones MQL5 (`SymbolInfo`, `CopyRates`, `iClose`, etc.). | Alta |
| **MT5-03** | Datos Macro y Futuros | El EA debe obtener datos de instrumentos macro (DXY, 10-Year Note, 30-Year Treasury, CRB Index) y futuros de divisas para análisis de Open Interest y COT. | Alta |

Soporte para Stop Entries, Limit Entries, órdenes híbridas, Day Trading entries, Scalping entries, **Mega Trade entries** y **Stock entries**. | Alta |
| **MT5-05** | Gestión de Posiciones | El EA debe modificar y cerrar posiciones mediante funciones MQL5 (`PositionModify`, `PositionClose`, `TradeAction`, etc.). Soporte para trailing stop dinámico, gestión de swing trading, gestión OSOK, gestión Day Trading, gestión Scalping, **gestión Mega Trades** y **gestión Stock Trading**. | Alta |

| **MT5-10** | Multi-Símbolo | El EA debe poder operar en múltiples símbolos desde un único gráfico (usando `SymbolSelect` y `CopyRates`). | Media |
| **MT5-11** | Optimización | El EA debe ser compatible con el "Strategy Tester" de MT5 para backtesting y optimización de parámetros. | Alta |
| **MT5-12** | Licencia y Activación | El EA debe incluir un sistema de licencia que valide la activación del producto, permitiendo diferenciar entre uso personal y comercial. | Alta |

##### Dominio Analysis: Módulo 1 - Detección de Setups ICT

| ID | Requisito | Descripción | Prioridad |
| :--- | :--- | :--- | :--- |
| **RF-001** | Detección de Order Blocks | El EA debe identificar automáticamente Order Blocks alcistas y bajistas. **Debe priorizar institutional order flow sobre patrones clásicos. La identificación debe incluir:** | Alta |
| | | - **Monthly Order Blocks:** Velas bajistas violadas al alza (OB alcista) y velas alcistas violadas a la baja (OB bajista). | |
| | | - **Sponsorship Validation:** Solo OB con Institutional Sponsorship (displacement + retorno a fair value). | |
| | | - **Open-to-Close Range:** Medir el rango open-to-close, no high-low total. | |
| | | - **Consecutive Candles:** Agrupar velas consecutivas en la misma dirección (3+ down = un solo bullish OB). | |
| **RF-002** | Detección de FVG | El EA debe identificar Fair Value Gaps (huecos de valor razonable) en todas las temporalidades configuradas. **La detección debe basarse en la estructura de 3 velas: vela izquierda (rango), vela central (direccional), vela derecha (confirmación).** | Alta |
| **RF-003** | Cálculo de Equilibrium | El EA debe calcular el punto de equilibrio (50%) de cualquier rango de precio identificado. **Incluir Fibonacci Grading para proyectar niveles clave dentro del rango total.** | Alta |
| **RF-004** | Zonas de Descuento y Prima | El EA debe clasificar el precio actual en relación al equilibrium: por debajo del 50% = zona de descuento (compra), por encima del 50% = zona de prima (venta). | Alta |
| **RF-005** | Optimal Trade Entry (OTE) | El EA debe calcular la zona OTE (62%-79% de retroceso) de cualquier rango de precio identificado y utilizarla como área de entrada preferente. | Alta |
| **RF-006** | Clasificación de Mercado | El EA debe clasificar automáticamente el estado del mercado en tiempo real: Expansión, Retracement, Reversal o Consolidación. | Alta |
| **RF-007** | Detección de Turtle Soup | El EA debe identificar patrones de falsa ruptura: Turtle Soup Long y Turtle Soup Short. | Alta |
| **RF-008** | Detección de Stop Runs / Liquidity Hunts | El EA debe identificar cuando el precio activa stops (buy stops o sell stops) y marcarlo como un evento de liquidez. | Alta |
| **RF-009** | Market Protraction / Judas Swing | El EA debe identificar movimientos de protracción (Judas Swings) basados en la hora del día. | Media |
| **RF-010** | Análisis Multi-Temporal | El EA debe analizar múltiples temporalidades simultáneamente. **Top-Down completo: Monthly → Weekly → Daily → H4 → H1 → M15 → M5 → M1. Modelos de trading configurables. Time of Day como filtro.** | Alta |

##### Dominio Execution: Módulo 3 - Gestión de Riesgo (Optimizado)

| **RF-020** | Stop Loss Dinámico | El EA debe calcular el stop loss basado en la estructura de mercado. **Debe permitir stops ultra ajustados (5-10 pips) basados en estructura de precio en M5 y detectar Lethargic Response como señal de salida.** | Alta |

| **RF-030** | Filtro de Días | El EA debe permitir configurar los días de la semana en los que está permitido operar (ej. evitar lunes y viernes). | Media |
| **RF-031** | Filtro de Volatilidad | El EA debe filtrar entradas cuando la volatilidad (rango ATR) supere un valor máximo configurable. | Media |
| **RF-052** | **Filtro de Ratio Recompensa/Riesgo (Optimizado)** | El EA debe filtrar setups que no cumplan con un ratio R:R mínimo configurable. **Valores: 3:1 (mínimo), 5:1 (óptimo). Para scalping: R:R 1:1.** *[Nota: Mega Trades movido a RF-906]* | Alta |
| **RF-053** | Objetivo de Crecimiento Mensual | El EA debe permitir configurar un objetivo de crecimiento mensual. **Opciones: 6%, 10% o personalizado.** | Media |
| **RF-054** | Salida Gradual (Scaling Out) | El EA debe implementar salida gradual en múltiples niveles de take profit: **TP1 (50% en 3:1), TP2 (25% en 5:1), TP3 (25% en 9:1+).** | Alta |
| **RF-055** | Límite de Operaciones por Semana | El EA debe permitir configurar un máximo de operaciones por semana (por defecto 3-5). **Priorizar calidad sobre cantidad (One Shot One Kill). Para day trading: 1-2 operaciones/día; para scalping: 3-5 operaciones/día.** | Media |
| **RF-056** | Priorización de Setups de Alta Probabilidad | El EA debe priorizar setups basados en niveles institucionales (Daily, H4). | Alta |
| **RF-057** | Objetivo Semanal de Pips | El EA debe permitir configurar un objetivo semanal de pips (por defecto 20 pips). | Media |
| **RF-119** | Evaluación de Dynamic Response | El EA debe evaluar la velocidad de respuesta del precio al llegar a un nivel de sponsorship como confirmación de entrada. **Incluir identificación de propulsion blocks (respuesta inmediata).** | Alta |
| **RF-154** | Gestión de Gap Risk | El EA debe considerar gap risk (reprice agresivo para cerrar FVG) en la gestión de stop loss y take profit. | Alta |
| **RF-156** | Utilización de Mean Threshold | El EA debe utilizar el mean threshold (50% del rango open-to-close) como nivel clave para entradas y stop loss. **Debe ser utilizado como stop loss ajustado en propulsion blocks.** | Alta |
| **RF-320** | Límite de Riesgo: 1% del 30% del Equity | El EA debe usar el 1% del 30% del equity total (0.3% del equity total) como riesgo máximo por operación de posición trading. | Alta |
| **RF-362** | Riesgo del 1.5% por Swing Trade | El EA debe usar el 1.5% del equity como riesgo máximo por operación de swing trading. | Alta |
| **RF-366** | Cálculo de R-Múltiples por Trade | El EA debe calcular y registrar el R-múltiple (R:R) de cada swing trade. | Media |
| **RF-367** | Simulación de Rendimiento Anual | El EA debe calcular el rendimiento esperado anual basado en win rate y avg RR de swing trades. | Media |
| **RF-510.1** | **Cálculo Genérico de ADR (NUEVO)** | El EA debe calcular el Average Daily Range de los últimos 5 días hábiles para todos los modelos. **Este cálculo será consumido por RF-510 (Day Trading) y RF-532 (Scalping).** | Alta |
| **RF-920** | **Riesgo Mega Trade (0.3%)** | El EA debe usar el 0.3% del equity como riesgo máximo por operación de Mega Trade. | Alta |
| **RF-921** | **Riesgo Stock Trading (1.5%)** | El EA debe usar el 1.5% del equity como riesgo máximo por operación de Stock Trading. | Alta |
